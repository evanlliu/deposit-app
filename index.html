本地模式

新增定存

填写本金、本金+利息、日期后自动计算。
开户汇率根据开户日期获取；结束汇率根据结束日期获取。周末/节假日通常会返回最近一个工作日汇率。
新增当前定存时可以不填；移入历史后可编辑补充。
说明：现在只需要手动录入开户日期、开始日期、结束日期、本金 TRY、本金+利息 TRY、年利率 TRY。利息、存款天数、剩余天数、CNY/USD 换算、CNY 年化收益率都会自动计算。结束日期如果是未来日期，系统会暂时使用最新可用汇率，等到结束日期后可以点击“刷新汇率”重新按开户日期和结束日期更新汇率。

同步设置

v24 起支持自动云端同步：API 地址和访问密码都会写入云端配置/备份 JSON。任意设备保存一次后，其他设备打开页面会自动读取云端配置和数据。你仍然可以在这里自定义 Worker 地址和访问密码。

汇率设置

开户汇率和结束汇率都会按这里选择的来源获取。TCMB 更适合 TRY 场景；Frankfurter 无需 Key；ExchangeRate-API 不填 Key 时只使用开放最新汇率；exchangerate.host 和 currencyapi.com 需要 API Key。exchangerate.host 会把当天获取到的最近一年汇率缓存到 GitHub 的 data.json；如果“获取最新 API 返回数据”选择 NO，则当天后续刷新优先使用 GitHub 缓存，避免重复消耗 API 次数；如果选择 YES，则每次刷新都会重新调用 API 并覆盖今天的缓存。默认情况下密钥只保存在当前浏览器本地配置；如果选择“同步 API Key 到云端”，会写入 GitHub 私有仓库的 data.json，换设备可自动带出。汇率值不再强制保留 4 位小数，会尽量保留接口返回的精度。

列配置

当前定存列配置

当前定存表和历史记录表独立保存配置。可以隐藏列，也可以修改显示列名；字段 code 不会变化。

历史记录列配置

历史记录可以单独修改列名，例如把“银行”改成“土耳其银行”，不会影响当前定存表。
“序号”列也可以隐藏/显示；“操作”列固定显示。列名配置会和列显示配置一起保存到云端，多设备同步。比如 code 仍然是 bank，但显示名称可以改成“土耳其银行”。

邮件提醒设置

点击变量，自动插入到标题或内容的光标位置 例如点击“银行”会插入 {{bank}}
说明：“利息 CNY”= 利息 TRY × 结束 TRY→CNY 汇率；“获得 CNY”= 本金+利息按结束汇率折算后减去本金按开户汇率折算后的结果，包含汇率变化影响。
点击“预览提醒邮件”。
邮件提醒规则会保存到云端。正式发送由 Cloudflare Worker + Resend 执行；邮件正文只发送“邮件内容模板”里的内容，不会再额外追加字段。你可以点击“模板变量”按钮插入字段，例如点击“银行”会插入 {{bank}}。提前几天推送可只填一个数字，例如 1 或 3；留空则不提前推送。到期当天每 X 小时推送留空则不循环推送；两个规则都填写时会同时生效。
