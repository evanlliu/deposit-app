定期存款管理系统

当前定存 + 历史记录；只需录入关键字段，汇率和收益自动计算。

刷新汇率会更新当前定存全部记录；历史记录只更新配置天数内的记录。
本地模式

新增定存

填写本金、本金+利息、日期后自动计算。
开户汇率根据开户日期获取；结束汇率根据结束日期获取。周末/节假日通常会返回最近一个工作日汇率。
新增当前定存时可以不填；移入历史后可编辑补充。
说明：现在只需要手动录入开户日期、开始日期、结束日期、本金 TRY、本金+利息 TRY、年利率 TRY。利息、存款天数、剩余天数、CNY/USD 换算、CNY 年化收益率都会自动计算。结束日期如果是未来日期，系统会暂时使用最新可用汇率，等到结束日期后可以点击“刷新汇率”重新按开户日期和结束日期更新汇率。

同步设置

不填写 API 地址时，系统使用本地模式，数据只保存在当前浏览器。填写后，手机和电脑会通过 GitHub 的 data.json 同步。API 地址会写入云端配置，方便换设备后识别；访问密码只保存在本机，不写入 data.json。

汇率设置

开户汇率和结束汇率都会按这里选择的来源获取。TCMB 更适合 TRY 场景；Frankfurter 无需 Key；ExchangeRate-API 不填 Key 时只使用开放最新汇率；exchangerate.host 和 currencyapi.com 需要 API Key。exchangerate.host 已支持 timeframe 批量获取历史汇率，刷新大量记录时会优先按日期区间缓存，减少 API 调用次数。默认情况下密钥只保存在当前浏览器本地配置；如果选择“同步 API Key 到云端”，会写入 GitHub 私有仓库的 data.json，换设备可自动带出。汇率值不再强制保留 4 位小数，会尽量保留接口返回的精度。

列配置

当前定存列配置

当前定存表和历史记录表独立保存配置。例如可以只隐藏当前定存的“本金(USD)”。

历史记录列配置

历史记录可以保留更多字段，例如删除时间、兑换原因、兑换金额等。
“序号”列现在也可以隐藏/显示；“操作”列固定显示，避免隐藏后无法编辑或恢复记录。列配置会保存在本地数据中，后续同步到云端时也会一起保存。

邮件提醒设置

点击变量，自动插入到标题或内容的光标位置 例如点击“银行”会插入 {{bank}}
说明：“利息 CNY”= 利息 TRY × 结束 TRY→CNY 汇率；“获得 CNY”= 本金+利息按结束汇率折算后减去本金按开户汇率折算后的结果，包含汇率变化影响。
点击“预览提醒邮件”。
邮件提醒规则会保存到云端。正式发送由 Cloudflare Worker + Resend 执行；邮件正文只发送“邮件内容模板”里的内容，不会再额外追加字段。你可以点击“模板变量”按钮插入字段，例如点击“银行”会插入 {{bank}}。提前几天推送可只填一个数字，例如 1 或 3；留空则不提前推送。到期当天每 X 小时推送留空则不循环推送；两个规则都填写时会同时生效。
